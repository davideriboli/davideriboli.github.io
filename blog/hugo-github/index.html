Hugo + GitHub Pages (sporco e veloce)

14 Nov 2020

Questo articolo era nato in inglese per un altro sito e sono troppo pigro per tradurlo di nuovo in italiano. Ma se non riuscite a capire che c’è scritto, è meglio che lasciate perdere: significa che è roba troppo complicata per voi.

Ok… here you can find the right way to do it. It’s the very right way. And it’s long. And hard. But it’s right… And this is my (quick and dirty) way.

3 - CHECK YOUR TOOLS

Before you start, I assume you:

  1. know what Hugo and GitHub Pages are;
  2. know how to use a terminal;
  3. have installed hugo and git on your sys.

2 - BUILD OFFLINE

Navigate in your sys where you want to store your work and make a new site with Hugo.

hugo new site MyShinySite

Cd in your new site and launch server.

 cd MyShinySite

hugo server --watch --disableFastRender

Complete your new site with template and contents.

Stop the Hugo server with CTRL + c (on Linux) or Right-CTRL + c (on Windows).

1 - PREPARE TAKE OFF

Login to GitHub and make a new repo, named yourusername.github.io. Add a correct README.md and the license you prefer.

Clone your new repo locally and launch a terminal where you want to store your repo.

git clone PutHereTheUrlOfYourRepo

(Optional) If you’re under a Linux distro, copy the README.md and License.md files to your Templates folder. I’ve added even a Post.md file for a quick kickstart of a new post.

Go in your Hugo site folder and create the static files with:

hugo

Now you have a new folder, named Public.

Go inside Public and add a README.md and a License.md (Linux guys here use the right click and the New File option; others can copy&paste from the cloned repo).

Copy all the files and folders inside your Public.

Go in your cloned repo. Trash all the files and folders you see (select them one by one; do not use select all ‘cause you could destroy the invisble .GIT folder, fucking up everything). Paste the files and folders you’ve copied from Public.

0 - TAKE OFF!

Open a terminal in your cloned repo folder.

git add *

git commit -a

git push origin master

Done: you’re site is flying!
