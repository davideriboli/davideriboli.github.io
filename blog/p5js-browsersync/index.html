P5js - Preparare l'ambiente di lavoro con BrowserSync Server

14 Nov 2020

Di solito preferisco SublimeText sia per lavorare in Processing che in P5, ma in quest’ultimo caso l’editing può diventare una vera spina nel culo senza un server veloce da cui avere un feedaback. La ricetta qui sotto funziona anche in Mac (la gente Mela trova qui come usare brew per node+npm) eLinux.

Installare Node.js via download regolare o con Choco/Brew.

choco install nodejs

Installare browser-sync via nodejs

npm install -g browser-sync

Creare una cartella di progetto P5.js con tutti i file necessari.

Aprire la cartella in un terminale e lanciare:

browser-sync start --server --directory --files "*"

Aggiungere il codice richiesto (quando richiesto) al’HTML index file del progetto.

Lanciare di nuovo:

browser-sync start --server --directory --files "*"

Fatto : )

Per fermare browser-sync: CTRL + C.

Altri comandi browser-sync: https://browsersync.io/docs/command-line.

Aggiornamento

Da quest’anno ho mollato SublimeText per VS Code. L’editor di Microsoft mi pare davvero - almeno per il momento - il miglior editor di codice che ci sia in giro e nel godzillione di estensioni che gli ruotano intorno ce ne sono alcune ottime per lavorare con P5js. Però Browser-Sync è davvero un buon software che può tornare utile per un sacco di roba e quindi l’articolo è rimasto.
